Assessoria Esportiva Elton Almeida

O projeto é estruturado com uma arquitetura Full Stack, utilizando Node.js no backend e React no frontend.

Abaixo estão detalhadas as principais tecnologias, bibliotecas e ferramentas utilizadas neste projeto.

1. Frontend (Interface do Usuário)

  • React 19 (react, react-dom): Framework principal para construção da interface.
  • React Router v7 (react-router-dom): Gerenciamento de rotas e navegação da aplicação.
  • Zustand (zustand): Gerenciador de estado global leve e moderno.
  • Recharts (recharts): Biblioteca de gráficos para exibição da evolução dos atletas.
  • Lucide React (lucide-react): Conjunto de ícones vetoriais modernos.
  • Vite 8 (vite, @vitejs/plugin-react): Ferramenta de build extremamente rápida para o desenvolvimento frontend.
  • PWA (Progressive Web App) (vite-plugin-pwa): Configurações para permitir que a aplicação seja instalada e funcione como um aplicativo nativo no celular ou desktop.

2. Backend (Servidor e APIs)

Node.js & Express 5 (express): Servidor HTTP e API REST.

  • Segurança:
    • Helmet (helmet): Configuração de cabeçalhos HTTP seguros.
    • Express Rate Limit (express-rate-limit): Proteção contra ataques de força bruta limitando requisições.
    • CORS (cors): Permissões de compartilhamento de recursos entre origens diferentes.
  • Autenticação:
    • JSON Web Tokens (jsonwebtoken): Geração e validação de tokens para rotas protegidas.
    • Bcrypt (bcrypt): Criptografia e hashing seguro de senhas.
  • Validação de dados: Express Validator (express-validator).
  • Upload de arquivos: Multer e Multer S3 para upload direto para o Amazon S3 usando a SDK oficial da AWS (@aws-sdk/client-s3).
  • Agendamento de tarefas: Node Cron (node-cron) para execução de rotas ou rotinas periódicas em background.
  • Automação: Puppeteer para renderização em navegador headless (Usado para gerar um certificado digital para o atleta).
  • Notificações: Web Push (web-push) para envio de notificações push direto nos navegadores dos atletas.
  • Monitoramento: Sentry (@sentry/node) para rastreamento de erros e monitoramento de performance em tempo real.

3. Banco de Dados e Migrações

  • PostgreSQL (pg): Banco de dados relacional utilizado.
  • Node PG Migrate (node-pg-migrate): Ferramenta para gerenciar e executar as migrações (migrations) estruturais do banco de dados PostgreSQL usando Javascript/SQL.

4. Infraestrutura e DevOps

  • Docker (Dockerfile): Containerização baseada em Alpine Linux com instalação prévia do Chromium para rodar o Puppeteer em ambiente isolado.
  • Easypanel: Scripts de inicialização configurados para publicação simplificada na plataforma Easypanel.

Sistema de Estoque simplificado

O projeto é estruturado com uma arquitetura robusta integrando Inteligência Artificial e automação de mensagens, focado em consultas e gerenciamento de estoque em tempo real.

Abaixo estão detalhadas as principais tecnologias, bibliotecas e ferramentas utilizadas neste projeto:

1. Frontend (Interface do Usuário)

  • React 19: Biblioteca JavaScript para construção da interface de usuário.
  • Vite: Ferramenta de build e servidor de desenvolvimento super rápido.
  • Lucide React: Pacote de ícones modernos e limpos.
  • CSS Vanilla: Estilização tradicional direta em CSS.

2. Backend (Servidor e APIs)

  • Node.js (v20): Ambiente de execução JavaScript no servidor.
  • Express (v5): Framework web para criação de rotas HTTP e gerenciamento do servidor.
  • gRPC (@grpc/grpc-js): Protocolo de comunicação de alta performance para chamadas de procedimentos remotos.
  • Prisma Client (v6): ORM (Object-Relational Mapping) utilizado para modelagem de banco de dados e consultas de forma segura e tipada.
  • Zod: Biblioteca de declaração de esquemas e validação de dados.
  • JWT (JSON Web Token) & BcryptJS: Usados para autenticação, controle de sessão e criptografia/hashing de senhas.
  • Multer: Middleware para tratamento de upload de arquivos (multipart/form-data).
  • PDF-Parse: Biblioteca para extração e processamento de dados contidos em arquivos PDF.

3. Inteligência Artificial (NVIDIA NIM APIs)

  • Meta Llama 3.1 8B Instruct: Modelo de linguagem (LLM) utilizado para processar e inferir as intenções de busca do usuário (como extrair nomes e códigos de produtos a partir de mensagens).
  • OpenAI Whisper Large v3 (via NVIDIA Riva gRPC): Utilizado para transcrever as mensagens de áudio/voz enviadas pelos usuários em português (pt-BR) para texto.

4. Integração do WhatsApp

  • Evolution API: Utilizada para conectar o fluxo do WhatsApp ao backend, gerenciando o envio e recebimento de mensagens e mídias, permitindo que os clientes consultem o estoque diretamente por conversa.

5. Banco de Dados e Infraestrutura

  • PostgreSQL: Banco de dados relacional.
  • Docker: Utilizado para empacotamento, distribuição e execução em contêineres através do (Dockerfile).
